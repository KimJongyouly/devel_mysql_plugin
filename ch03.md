# 제3장: 집계 UDF — GROUP BY와 함께 중앙값 계산

---

## 3.1 집계 UDF란

### 스칼라 UDF와의 차이

2장까지 만든 UDF는 **스칼라(scalar) UDF**입니다. 입력 행 하나에 결과 하나를 반환합니다.

```sql
SELECT helloworld(name) FROM users;
-- 행마다 한 번씩 호출됨
```

반면 **집계(aggregate) UDF**는 여러 행을 하나로 묶어 단일 결과를 반환합니다. `SUM()`, `AVG()`, `COUNT()`와 같은 내장 집계 함수와 동일한 방식으로 동작합니다.

```sql
SELECT my_median(score) FROM exam_results;
-- 전체 행을 처리한 뒤 하나의 값 반환

SELECT department, my_median(salary) FROM employees GROUP BY department;
-- 부서별로 묶어 각 그룹의 중앙값 반환
```

MySQL에는 `MEDIAN()` 함수가 기본 내장되어 있지 않습니다. 이것이 집계 UDF가 필요한 대표적인 사례입니다.

### 중앙값(Median)이란

중앙값은 전체 데이터를 오름차순으로 정렬했을 때 정중앙에 위치한 값입니다.

- 홀수 개: 정중앙 값
- 짝수 개: 중앙 두 값의 평균

예:
- `[1, 3, 5]` → 중앙값 = **3**
- `[1, 3, 5, 7]` → 중앙값 = **(3 + 5) / 2 = 4.0**

평균과 달리 극단값(이상치)에 영향을 덜 받아, 연봉·부동산 가격·응답 시간 등의 분포 분석에 자주 사용됩니다.

## 3.2 집계 UDF의 6단계 라이프사이클

집계 UDF는 스칼라 UDF의 3개 함수(`_init`, 메인, `_deinit`)보다 많은 6개의 함수로 구성됩니다.

```
쿼리 시작
    ↓
my_median_init()          ← 1회: 메모리 할당, 인자 검증
    ↓
[그룹 1 시작]
my_median_clear()         ← 각 그룹 시작 시: 누적값 초기화
my_median_reset()         ← 그룹의 첫 번째 행: clear 후 첫 값 추가
my_median_add()           ← 그룹의 두 번째~ 행: 값 누적
my_median()               ← 그룹 종료: 결과 계산 및 반환
    ↓
[그룹 2 시작]
my_median_clear()         ← 이전 그룹 데이터 초기화
my_median_reset()
my_median_add() × n
my_median()
    ↓
(그룹마다 반복)
    ↓
my_median_deinit()        ← 1회: 메모리 해제
```

| 함수 | 시그니처 형태 | 호출 횟수 |
|------|-------------|---------|
| `_init()` | `bool f(UDF_INIT*, UDF_ARGS*, char*)` | 1회 |
| `_deinit()` | `void f(UDF_INIT*)` | 1회 |
| `_clear()` | `void f(UDF_INIT*, uchar*, uchar*)` | 그룹당 1회 |
| `_reset()` | `void f(UDF_INIT*, UDF_ARGS*, uchar*, uchar*)` | 그룹당 1회 |
| `_add()` | `void f(UDF_INIT*, UDF_ARGS*, uchar*, uchar*)` | 그룹의 (행 수 - 1)회 |
| 메인 함수 | `double f(UDF_INIT*, UDF_ARGS*, uchar*, uchar*)` | 그룹당 1회 |

### `_clear()`가 없으면 어떻게 될까

`GROUP BY`를 사용할 때 `_clear()`가 없으면 이전 그룹의 데이터가 다음 그룹에 그대로 누적됩니다.

```
그룹 A: [10, 20, 30]
그룹 B: [100, 200]

_clear() 없을 때:
- 그룹 A 처리 후 values = [10, 20, 30]
- 그룹 B에서 add → values = [10, 20, 30, 100, 200]
- my_median() → 30 (잘못된 결과!)

_clear() 있을 때:
- 그룹 B 시작 시 values = [] (초기화)
- 그룹 B에서 add → values = [100, 200]
- my_median() → 150 (올바른 결과)
```

## 3.3 구현: `my_median()` 집계 UDF

### 컨텍스트 구조체 설계

집계 UDF는 그룹 내 누적 데이터를 저장할 공간이 필요합니다. 이를 위해 커스텀 구조체를 정의하고 `initid->ptr`에 저장합니다.

```cpp
struct MedianContext {
    std::vector<double>* values;
};
```

`std::vector<double>`를 사용해 그룹 내 값들을 동적으로 누적합니다.

### 전체 코드 (macOS/MySQL 8.0)

```cpp
#include <mysql/udf_registration_types.h>
#include <vector>
#include <algorithm>
#include <cstring>
#include <new>

struct MedianContext {
    std::vector<double>* values;
};

extern "C" {

// 1. 초기화: 메모리 할당, 인자 검증
bool my_median_init(UDF_INIT* initid, UDF_ARGS* args, char* message) {
    if (args->arg_count != 1) {
        strcpy(message, "my_median() requires exactly one argument");
        return true;
    }
    if (args->arg_type[0] != REAL_RESULT &&
        args->arg_type[0] != INT_RESULT &&
        args->arg_type[0] != DECIMAL_RESULT) {
        strcpy(message, "my_median() requires a numeric argument");
        return true;
    }

    // MySQL에게 인자를 double로 변환해달라고 요청
    args->arg_type[0] = REAL_RESULT;

    MedianContext* ctx = new (std::nothrow) MedianContext();
    if (!ctx) {
        strcpy(message, "my_median(): out of memory");
        return true;
    }
    ctx->values = new (std::nothrow) std::vector<double>();
    if (!ctx->values) {
        delete ctx;
        strcpy(message, "my_median(): out of memory");
        return true;
    }

    initid->ptr        = reinterpret_cast<char*>(ctx);
    initid->maybe_null = true;
    initid->const_item = false;
    return false;
}

// 2. 종료: 메모리 해제
void my_median_deinit(UDF_INIT* initid) {
    MedianContext* ctx = reinterpret_cast<MedianContext*>(initid->ptr);
    if (ctx) {
        delete ctx->values;
        delete ctx;
        initid->ptr = nullptr;
    }
}

// 3. 그룹 초기화: 이전 그룹 데이터 삭제
void my_median_clear(UDF_INIT* initid, unsigned char* is_null, unsigned char* error) {
    MedianContext* ctx = reinterpret_cast<MedianContext*>(initid->ptr);
    ctx->values->clear();
    *is_null = 0;
}

// 4. 첫 번째 행 처리
void my_median_reset(UDF_INIT* initid, UDF_ARGS* args,
                     unsigned char* is_null, unsigned char* error) {
    my_median_clear(initid, is_null, error);
    if (args->args[0]) {
        MedianContext* ctx = reinterpret_cast<MedianContext*>(initid->ptr);
        ctx->values->push_back(*reinterpret_cast<double*>(args->args[0]));
    }
}

// 5. 두 번째~ 행 처리: 값 누적
void my_median_add(UDF_INIT* initid, UDF_ARGS* args,
                   unsigned char* is_null, unsigned char* error) {
    if (args->args[0]) {
        MedianContext* ctx = reinterpret_cast<MedianContext*>(initid->ptr);
        ctx->values->push_back(*reinterpret_cast<double*>(args->args[0]));
    }
}

// 6. 결과 계산: 정렬 후 중앙값 반환
double my_median(UDF_INIT* initid, UDF_ARGS* args,
                 unsigned char* is_null, unsigned char* error) {
    MedianContext* ctx = reinterpret_cast<MedianContext*>(initid->ptr);

    if (ctx->values->empty()) {
        *is_null = 1;
        return 0.0;
    }

    std::vector<double>& v = *(ctx->values);
    size_t n = v.size();
    std::sort(v.begin(), v.end());

    if (n % 2 == 0) {
        return (v[n / 2 - 1] + v[n / 2]) / 2.0;
    } else {
        return v[n / 2];
    }
}

} // extern "C"
```

### 코드 상세 분석

#### `_init()`: 타입 강제 변환

```cpp
args->arg_type[0] = REAL_RESULT;
```

`_init()`에서 `arg_type`을 `REAL_RESULT`로 덮어쓰면, MySQL은 정수나 DECIMAL 타입의 인자를 자동으로 `double`로 변환해서 넘겨줍니다. 덕분에 메인 로직에서는 타입 분기 없이 항상 `double*`로 캐스팅할 수 있습니다.

#### `new (std::nothrow)` 관용구

```cpp
MedianContext* ctx = new (std::nothrow) MedianContext();
if (!ctx) { ... }
```

표준 `new`는 메모리 부족 시 `std::bad_alloc` 예외를 던집니다. UDF 내부에서 예외가 전파되면 MySQL 서버가 크래시될 수 있습니다. `std::nothrow` 버전은 실패 시 예외 대신 `nullptr`을 반환하므로 UDF에서는 이 방식이 더 안전합니다.

#### `_reset()` = `_clear()` + `_add()` 첫 번째

```cpp
void my_median_reset(UDF_INIT* initid, UDF_ARGS* args,
                     unsigned char* is_null, unsigned char* error) {
    my_median_clear(initid, is_null, error);   // 초기화
    if (args->args[0]) {                        // 첫 번째 값 추가
        ctx->values->push_back(...);
    }
}
```

`_reset()`은 각 그룹의 **첫 번째 행**에만 호출됩니다. 내부적으로 `_clear()`를 호출해 이전 데이터를 비운 뒤 첫 값을 추가합니다. 두 번째 행부터는 `_add()`가 호출됩니다.

#### `_add()`: double 포인터 역참조

```cpp
ctx->values->push_back(*reinterpret_cast<double*>(args->args[0]));
```

`REAL_RESULT` 타입의 인자는 `args->args[0]`이 `double*` 포인터입니다. `reinterpret_cast<double*>`로 캐스팅한 뒤 `*`로 역참조해 `double` 값을 얻습니다.

#### 메인 함수: 중앙값 알고리즘

```cpp
std::sort(v.begin(), v.end());  // O(n log n)

if (n % 2 == 0) {
    return (v[n / 2 - 1] + v[n / 2]) / 2.0;  // 짝수: 중간 두 값의 평균
} else {
    return v[n / 2];  // 홀수: 정중앙 값
}
```

예시:
- `n = 5`, `v = [1, 1, 3, 4, 5]` → `v[2] = 3`
- `n = 6`, `v = [1, 1, 3, 4, 5, 10]` → `(v[2] + v[3]) / 2 = (3 + 4) / 2 = 3.5`

**시간 복잡도**: O(n log n) — 정렬이 지배

**공간 복잡도**: O(n) — 그룹 내 모든 값을 메모리에 유지

그룹이 수백만 행이면 메모리 사용량이 커질 수 있습니다. 이 구현은 정확한 중앙값을 위해 전체 데이터를 메모리에 올리는 방식입니다.

## 3.5 GROUP BY와 함께 사용하기

### 전체 데이터 중앙값

```sql
-- 테스트 데이터
CREATE TABLE test_scores (score DOUBLE);
INSERT INTO test_scores (score) VALUES (10), (20), (30), (40), (50);

SELECT my_median(score) FROM test_scores;
-- +------------------+
-- | my_median(score) |
-- +------------------+
-- |               30 |
-- +------------------+

-- 값 추가 후 짝수 개
INSERT INTO test_scores (score) VALUES (35);
SELECT my_median(score) FROM test_scores;
-- +------------------+
-- | my_median(score) |
-- +------------------+
-- |             32.5 |   ← (30 + 35) / 2
-- +------------------+
```

### GROUP BY와 함께

```sql
CREATE TABLE exam_results (
    department VARCHAR(10),
    score DOUBLE
);

INSERT INTO exam_results VALUES
    ('A', 3), ('A', 1), ('A', 4), ('A', 1), ('A', 5),
    ('B', 10), ('B', 20), ('B', 30);

SELECT department, my_median(score) AS median_score
FROM exam_results
GROUP BY department;
-- +------------+--------------+
-- | department | median_score |
-- +------------+--------------+
-- | A          |            3 |  ← [1,1,3,4,5] → 3
-- | B          |           20 |  ← [10,20,30] → 20
-- +------------+--------------+
```

### 집계 UDF와 일반 집계의 조합

```sql
SELECT department,
       COUNT(*) AS cnt,
       AVG(score) AS avg_score,
       my_median(score) AS median_score
FROM exam_results
GROUP BY department;
```

집계 UDF는 내장 집계 함수(`COUNT`, `AVG`, `SUM`)와 동일한 쿼리에서 함께 사용할 수 있습니다.

### 빈 그룹 처리

```sql
SELECT my_median(score) FROM test_scores WHERE score > 9999;
-- +------------------+
-- | my_median(score) |
-- +------------------+
-- | NULL             |   ← 빈 그룹은 NULL 반환
-- +------------------+
```

`initid->maybe_null = true`와 `*is_null = 1` 설정 덕분에 빈 그룹에서 안전하게 NULL을 반환합니다.

## 3장 정리

- 집계 UDF는 `_init`, `_clear`, `_reset`, `_add`, 메인, `_deinit` 6개 함수로 구성된다.
- `_clear()`는 `GROUP BY` 사용 시 그룹마다 누적값을 초기화하는 핵심 함수다. 빠뜨리면 잘못된 결과가 나온다.
- `initid->ptr`에 커스텀 컨텍스트 구조체를 저장해 그룹 간 데이터를 유지한다.
- 등록 시 `CREATE AGGREGATE FUNCTION`을 사용한다 (`CREATE FUNCTION`이 아님).
- `new (std::nothrow)` 를 사용해 예외 없이 메모리 할당 실패를 처리한다.
