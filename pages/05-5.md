# 05-5. 빌드 시스템

### 왜 MySQL 소스가 필요한가

MySQL/MariaDB 바이너리 배포판에는 `mysql/plugin.h`, `handler.h`, `my_global.h` 등의 서버 내부 헤더가 포함되지 않습니다. 스토리지 엔진 개발을 위해서는 서버 소스코드에서 헤더를 가져와야 합니다.

중요한 제약: **서버 소스 버전이 설치된 MySQL/MariaDB 서버 버전과 정확히 일치해야 합니다.** 버전이 다르면 `INSTALL PLUGIN` 시 아래 오류가 발생합니다.

```
ERROR: API version for STORAGE ENGINE plugin is too different
```

### 선택적 빌드 방식

전체 MySQL 서버를 컴파일하는 것은 매우 오랜 시간이 걸립니다. `build.sh`는 필요한 부분만 선택적으로 빌드합니다.

```
cmake configure   → 헤더 경로·플래그 생성만 (빌드 없음)
        ↓
make GenError     → mysqld_error.h 등 생성 헤더만 빌드
        ↓
make (build.make) → ha_parquet.cc.o 만 컴파일
        ↓
링크              → Arrow + Parquet 라이브러리만 링크
                  → MySQL 심볼은 INSTALL PLUGIN 시 mysqld가 제공
        ↓
ha_parquet.so / ha_read_parquet.so
```

`GenError` 타겟만 빌드하는 것이 핵심입니다. MySQL 서버 빌드 시스템이 `mysqld_error.h` 같은 생성 헤더를 만들어 주는데, 이것이 없으면 컴파일이 실패합니다.

### macOS 빌드

```bash
cd 05_ReadParquet/Mac

# 1단계: 의존성 설치
brew install apache-arrow openssl@3 lz4 cmake

# 2단계: MySQL 서버 소스 클론
git clone --depth=1 https://github.com/mysql/mysql-server.git
mysql_ver=$(/usr/local/mysql/bin/mysql_config --version)
git -C mysql-server fetch --tags
git -C mysql-server checkout mysql-${mysql_ver}

# 3단계: 빌드
./build.sh
```

성공 시 `ha_read_parquet.so` 생성.

### Linux 빌드

```bash
cd 05_ReadParquet/Linux

# 1단계: 의존성 설치
sudo apt install libarrow-dev libparquet-dev cmake g++ libssl-dev

# 2단계: MariaDB 소스 클론
git clone --depth=1 https://github.com/MariaDB/server.git mariadb-server
mariadb_ver=$(mariadb_config --version | cut -d- -f1)
git -C mariadb-server checkout mariadb-${mariadb_ver}

# 3단계: 빌드
./build.sh
```

성공 시 `ha_parquet.so` 생성.
