# 05-6. 사용 방법

### 플러그인 설치

```sql
-- macOS/MySQL
INSTALL PLUGIN read_parquet SONAME 'ha_read_parquet.so';

-- Linux/MariaDB
INSTALL PLUGIN read_parquet SONAME 'ha_parquet.so';

-- 설치 확인
SHOW PLUGINS;
```

### 테이블 생성

```sql
CREATE TABLE sales (
    order_id   INT,
    product    VARCHAR(128),
    amount     DOUBLE,
    order_date DATE
) ENGINE=READ_PARQUET
  COMMENT='/data/warehouse/sales_2024.parquet';
```

컬럼 타입은 parquet 파일의 스키마와 호환되어야 합니다. 컬럼 **순서**가 일치해야 합니다 (이름 기반 매핑 아님).

### 테스트용 Parquet 파일 생성

Python으로 간단히 테스트 파일을 만들 수 있습니다.

```python
import pandas as pd

df = pd.DataFrame({
    'order_id':   [1, 2, 3],
    'product':    ['Apple', 'Banana', 'Cherry'],
    'amount':     [10.5, 20.0, 15.75],
    'order_date': pd.to_datetime(['2024-01-01', '2024-01-02', '2024-01-03'])
})
df.to_parquet('/tmp/sales.parquet', index=False)
```

### 쿼리

```sql
SELECT * FROM sales;
-- +-----------+---------+--------+------------+
-- | order_id  | product | amount | order_date |
-- +-----------+---------+--------+------------+
-- |         1 | Apple   |   10.5 | 2024-01-01 |
-- |         2 | Banana  |   20.0 | 2024-01-02 |
-- |         3 | Cherry  | 15.75  | 2024-01-03 |
-- +-----------+---------+--------+------------+

-- 집계
SELECT product, SUM(amount) AS total FROM sales GROUP BY product;

-- 다른 엔진의 테이블과 조인
SELECT s.product, c.category
FROM sales s JOIN product_catalog c ON s.product = c.name;
```

### 제거

```sql
DROP TABLE sales;
-- .prq 사이드카 파일도 자동 삭제됩니다

UNINSTALL PLUGIN read_parquet;
```

### 제약사항

| 항목 | 내용 |
|------|------|
| 쓰기 | 불가 — INSERT/UPDATE/DELETE 모두 `HA_ERR_WRONG_COMMAND` |
| 인덱스 | 미지원 — 항상 풀스캔 |
| 트랜잭션 | 미지원 |
| 컬럼 매핑 | 이름이 아닌 순서 기준 |
| 메모리 | 파일 전체를 메모리에 로드 |
| 버전 | MySQL/MariaDB 소스와 서버 버전이 일치해야 함 |
