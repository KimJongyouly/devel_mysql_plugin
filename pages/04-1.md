# 04-1. 데몬 플러그인이란

### UDF와 다른 플러그인 세계

앞의 세 장에서는 SQL 함수로 호출하는 UDF를 만들었습니다. UDF는 쿼리가 들어올 때만 실행됩니다. **데몬 플러그인(Daemon Plugin)** 은 다릅니다. MySQL 서버가 시작될 때 함께 시작되어, 서버가 종료될 때까지 백그라운드에서 독립적으로 실행됩니다.

데몬 플러그인의 대표적인 활용 사례:

- **리소스 모니터링**: CPU, 메모리, 디스크 I/O를 주기적으로 로그에 기록
- **백그라운드 정리 작업**: 만료된 세션, 임시 파일을 주기적으로 삭제
- **외부 시스템 연동**: 메트릭을 외부 모니터링 시스템(Prometheus 등)으로 주기적 전송
- **감사 로그**: 쿼리 통계를 비동기적으로 기록

### MySQL 플러그인 종류

| 플러그인 타입 | 상수 | 설명 |
|------------|------|------|
| UDF | (별도 등록) | SQL 함수 확장 |
| Daemon | `MYSQL_DAEMON_PLUGIN` | 백그라운드 작업 |
| Storage Engine | `MYSQL_STORAGE_ENGINE_PLUGIN` | 스토리지 계층 |
| Full-text Parser | `MYSQL_FTPARSER_PLUGIN` | 전문 검색 파서 |
| Audit | `MYSQL_AUDIT_PLUGIN` | 감사 로그 |
| Authentication | `MYSQL_AUTHENTICATION_PLUGIN` | 인증 확장 |

### UDF와의 구조적 차이

| 항목 | UDF | 데몬 플러그인 |
|------|-----|------------|
| 등록 방법 | `CREATE FUNCTION` | `INSTALL PLUGIN` |
| 실행 트리거 | SQL 호출 시 | 플러그인 설치 즉시 |
| 실행 공간 | SQL 실행 컨텍스트 | 별도 백그라운드 스레드 |
| 헤더 파일 | `mysql.h` | `mysql.h` + `mysql/plugin.h` |
| 플러그인 등록 | (없음, `_init/_deinit` 직접 내보내기) | `mysql_declare_plugin` 매크로 |
