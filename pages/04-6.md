# 04-6. 빌드와 설치

### Linux 빌드

```bash
cd 04_OS_logger/Linux
make
make install
```

Makefile은 MariaDB 11.4.3의 헤더를 직접 지정합니다:

```makefile
MARIADB_BASE = /home/mariadb/mariadb_11.4.3
CXXFLAGS = -fPIC -Wall -DMYSQL_DYNAMIC_PLUGIN \
           -I$(MARIADB_BASE)/include/server/mysql \
           -I$(MARIADB_BASE)/include/mysql/server
LDFLAGS  = -shared -lpthread
```

### macOS 빌드

macOS 버전은 MySQL 서버 소스 헤더(`mysql/plugin.h`)가 필요합니다.

```bash
cd 04_OS_logger/Mac
make setup   # GitHub에서 MySQL include/ 디렉터리만 sparse-checkout
make         # 컴파일
sudo make install
```

### 플러그인 설치

```sql
-- Linux/MariaDB
INSTALL PLUGIN os_logger SONAME 'os_logger.so';

-- macOS/MySQL
INSTALL PLUGIN my_resource SONAME 'my_resource.so';

-- 설치 확인
SELECT PLUGIN_NAME, PLUGIN_STATUS
FROM information_schema.PLUGINS
WHERE PLUGIN_NAME IN ('os_logger', 'my_resource');
```

### 동작 확인

Linux:

```bash
tail -f /home/mariadb/mariadb_11.4.3/data/*.err
# [os_logger] CPU Load: 0.23, RAM: 3412/7982 MB
```

macOS:

```bash
sudo tail -f /usr/local/mysql/data/*.err
# [Note] my_resource: CPU=4.2% | MEM=7821/16384MB | DISK_R=12KB/s ...
```

### 플러그인 제거

```sql
UNINSTALL PLUGIN os_logger;
-- deinit()이 호출되어 백그라운드 스레드가 안전하게 종료됩니다
```

### 서버 시작 시 자동 로드

`my.cnf`에 추가하면 MySQL/MariaDB 시작 시 자동으로 로드됩니다:

```ini
[mysqld]
plugin-load-add = os_logger=os_logger.so
```
