# 04-3. 백그라운드 스레드 구현

### Linux 버전 전체 코드

Linux의 `os_logger`는 `sysinfo()` 시스템 콜로 CPU 부하와 메모리 사용량을 측정합니다.

```cpp
#include <mysql.h>
#include <mysql/plugin.h>
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <pthread.h>
#include <sys/sysinfo.h>

static pthread_t logger_thread;
static volatile int stop_logger = 0;
static int thread_running = 0;

static void *os_logger_worker(void *p) {
    struct sysinfo info;
    while (!stop_logger) {
        if (sysinfo(&info) == 0) {
            double load = info.loads[0] / 65536.0;
            unsigned long total_ram = (info.totalram * info.mem_unit) / 1024 / 1024;
            unsigned long free_ram  = (info.freeram  * info.mem_unit) / 1024 / 1024;
            fprintf(stderr, "[os_logger] CPU Load: %.2f, RAM: %lu/%lu MB\n",
                    load, free_ram, total_ram);
        }
        for (int i = 0; i < 5 && !stop_logger; i++) sleep(1);
    }
    return NULL;
}

static int os_logger_init(void *p) {
    stop_logger    = 0;
    thread_running = 0;

    int ret = pthread_create(&logger_thread, NULL, os_logger_worker, NULL);
    if (ret != 0) {
        fprintf(stderr, "[os_logger] pthread_create failed: %d\n", ret);
        return 1;
    }
    thread_running = 1;
    return 0;
}

static int os_logger_deinit(void *p) {
    stop_logger = 1;
    if (thread_running) {
        pthread_join(logger_thread, NULL);
        thread_running = 0;
    }
    return 0;
}
```

### 코드 상세 분석

#### 전역 변수와 `volatile`

- `logger_thread`: 생성된 스레드의 ID를 저장합니다. `pthread_join()`에서 사용합니다.
- `stop_logger`: 메인 스레드(deinit)가 워커 스레드에게 종료 신호를 보내는 플래그입니다. `volatile`을 붙여 컴파일러가 이 변수를 레지스터에 캐시하지 않도록 합니다.

> **`volatile`의 역할**: 컴파일러는 최적화를 위해 반복 루프에서 변하지 않는 변수를 레지스터에 캐시합니다. `stop_logger`는 다른 스레드에서 변경되므로, `volatile` 없이는 워커 스레드가 갱신된 값을 읽지 못할 수 있습니다.

#### 워커 스레드 종료 전략

```cpp
for (int i = 0; i < 5 && !stop_logger; i++) sleep(1);
```

단순히 `sleep(5)`를 쓰면 `deinit()`이 호출됐을 때 최대 5초를 기다려야 합니다. 위 코드는 1초씩 5번 나눠 자면서 매 초마다 `stop_logger` 플래그를 확인합니다. `deinit()`이 호출되면 최대 1초 안에 반응합니다.

#### 안전한 종료: `pthread_join()`

```cpp
static int os_logger_deinit(void *p) {
    stop_logger = 1;           // 종료 신호
    if (thread_running) {
        pthread_join(logger_thread, NULL);  // 완전히 끝날 때까지 대기
        thread_running = 0;
    }
    return 0;
}
```

`UNINSTALL PLUGIN` 이후 플러그인 코드는 메모리에서 언로드됩니다. 만약 워커 스레드가 아직 실행 중인 상태에서 코드가 언로드되면 **세그폴트(segfault)** 가 발생합니다. `pthread_join()`으로 스레드가 완전히 종료됨을 보장해야 합니다.
