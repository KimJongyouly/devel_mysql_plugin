# 01-5. 동작 확인

### 기본 호출

```sql
SELECT helloworld();
-- 결과:
-- +------------------------------+
-- | helloworld()                 |
-- +------------------------------+
-- | Hello World Mysql Plugin-UDF |
-- +------------------------------+
```

### 테이블과 함께 사용

이 UDF는 행마다 같은 값을 반환하므로, 테이블과 함께 사용하면 모든 행에 동일한 값이 나옵니다.

```sql
CREATE TABLE t (id INT);
INSERT INTO t VALUES (1), (2), (3);

SELECT id, helloworld() FROM t;
-- +----+------------------------------+
-- | id | helloworld()                 |
-- +----+------------------------------+
-- |  1 | Hello World Mysql Plugin-UDF |
-- |  2 | Hello World Mysql Plugin-UDF |
-- |  3 | Hello World Mysql Plugin-UDF |
-- +----+------------------------------+
```

MySQL은 `initid->const_item = 1` 설정을 알고 있기 때문에, 실제로는 `helloworld()` 를 한 번만 호출하고 결과를 재사용하는 최적화를 수행할 수 있습니다.

### 잘못된 호출 확인

인자를 전달하면 `_init()` 에서 설정한 오류 메시지가 반환됩니다.

```sql
SELECT helloworld('test');
-- ERROR 1123 (HY000): Can't initialize function 'helloworld';
-- helloworld() requires no arguments
```

### 함수 제거

```sql
DROP FUNCTION IF EXISTS helloworld;
```

### 트러블슈팅

**`mysql.h: No such file or directory`**

```
helloworld_udf.cpp:1:10: fatal error: mysql.h: No such file or directory
```

MySQL 헤더가 없거나 `mysql_config` 경로가 올바르지 않습니다.

```bash
# mysql_config 위치 확인
which mysql_config
/usr/local/mysql/bin/mysql_config   # macOS 기본

# Makefile의 MYSQL_CONFIG 경로를 수정합니다
```

**`Can't open shared library 'helloworld_udf.so'`**

플러그인 파일이 올바른 디렉터리에 없거나 권한이 맞지 않습니다.

```sql
-- 플러그인 디렉터리 확인
SHOW VARIABLES LIKE 'plugin_dir';
```

해당 경로에 `.so` 파일이 있는지 확인하고, 읽기 권한이 있는지 점검합니다.

**`function 'helloworld' already exists`**

이미 등록된 함수를 다시 등록하려 했습니다.

```sql
DROP FUNCTION IF EXISTS helloworld;
-- 재등록
CREATE FUNCTION helloworld RETURNS STRING SONAME 'helloworld_udf.so';
```

플러그인 파일을 교체한 경우에도 반드시 `DROP FUNCTION` 후 재등록이 필요합니다. MySQL은 한 번 로드한 라이브러리를 캐시하기 때문에, 파일만 교체해도 새 코드가 즉시 반영되지 않습니다.
