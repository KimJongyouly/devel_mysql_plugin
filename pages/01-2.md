# 01-2. 개발 환경 준비

### macOS 환경 설정

macOS에서는 MySQL 공식 배포판과 Xcode Command Line Tools가 필요합니다.

**MySQL 8.0 설치:**

```bash
# Homebrew를 사용하는 경우
brew install mysql

# 또는 MySQL 공식 사이트에서 .pkg 설치
# 설치 경로: /usr/local/mysql
```

**Xcode Command Line Tools 설치:**

```bash
xcode-select --install
```

**설치 확인:**

```bash
/usr/local/mysql/bin/mysql_config --version
# 예: 8.0.43
```

`mysql_config`는 UDF 빌드 시 필요한 헤더 경로와 링커 플래그를 자동으로 알려주는 도구입니다. Makefile에서 아래처럼 활용합니다.

```bash
mysql_config --cflags   # 컴파일 플래그 (헤더 경로 포함)
mysql_config --plugindir  # 플러그인 디렉터리 경로
```

### Linux 환경 설정

Ubuntu 환경에서는 MariaDB 개발 헤더와 g++이 필요합니다.

**MariaDB 및 개발 파일 설치:**

```bash
sudo apt update
sudo apt install mariadb-server libmariadb-dev g++ make
```

**MariaDB 11.4.x를 직접 설치한 경우:**

```bash
# mysql_config 경로 확인
/home/mariadb/mariadb_11.4.3/bin/mysql_config --version
```

**설치 확인:**

```bash
g++ --version
mysql_config --cflags
```

### 헤더 파일 확인

UDF를 작성하려면 `mysql.h` 헤더가 핵심입니다. 이 파일은 `mysql_config --include` 경로에 있습니다.

```bash
mysql_config --include
# 예: -I/usr/local/mysql/include
```

`mysql.h` 를 열어보면 UDF 개발에 필요한 구조체들이 정의되어 있습니다.

```c
// mysql.h 내 UDF 관련 핵심 타입
typedef struct st_udf_init {
    my_bool   maybe_null;    // NULL 반환 가능 여부
    unsigned int  decimals;  // 소수점 자릿수 (REAL 반환시)
    unsigned long max_length; // 최대 반환 길이
    char     *ptr;           // 초기화 함수에서 할당한 메모리 포인터
    my_bool   const_item;    // 입력이 상수면 1
    void     *extension;
} UDF_INIT;

typedef struct st_udf_args {
    unsigned int  arg_count;    // 인자 개수
    enum Item_result *arg_type; // 각 인자의 타입
    char        **args;         // 각 인자의 값
    unsigned long *lengths;     // 각 인자의 길이
    char        *maybe_null;    // 각 인자의 NULL 가능 여부
    // ...
} UDF_ARGS;
```

이 두 구조체는 앞으로 모든 UDF 코드에서 반복해서 만나게 됩니다.
