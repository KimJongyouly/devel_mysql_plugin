# 01-3. 첫 번째 UDF 작성

### UDF의 세 가지 함수

MySQL 스칼라 UDF는 세 개의 C 함수로 구성됩니다. 함수 이름은 **UDF 이름을 접두어**로 합니다.

| 함수 | 시그니처 | 호출 시점 |
|------|---------|---------|
| `{name}_init()` | `bool f(UDF_INIT*, UDF_ARGS*, char*)` | 쿼리 실행 시작 시 **1회** |
| `{name}()` | `char* f(UDF_INIT*, UDF_ARGS*, char*, ulong*, char*, char*)` | **행마다** |
| `{name}_deinit()` | `void f(UDF_INIT*)` | 쿼리 완료 후 **1회** |

MySQL이 UDF를 호출할 때 이 순서는 항상 고정되어 있습니다.

```
쿼리 시작 → helloworld_init() 1회
              ↓
행 1     → helloworld()
행 2     → helloworld()
행 3     → helloworld()
              ↓
쿼리 종료 → helloworld_deinit() 1회
```

### extern "C" 선언

C++로 UDF를 작성할 때는 `extern "C"` 블록이 필수입니다. C++는 함수 오버로딩을 지원하기 위해 함수 이름을 내부적으로 변환(name mangling)합니다. 하지만 MySQL은 `.so` 파일에서 C 이름 규칙으로 함수를 찾습니다. `extern "C"` 없이 컴파일하면 MySQL이 함수를 찾지 못해 등록이 실패합니다.

```cpp
#ifdef __cplusplus
extern "C" {
#endif

// UDF 함수들은 여기에

#ifdef __cplusplus
}
#endif
```

`#ifdef __cplusplus` 조건부 컴파일을 사용하면 C와 C++ 양쪽에서 같은 소스를 컴파일할 수 있습니다.

### 전체 코드 설명

아래는 `helloworld` UDF의 전체 소스코드입니다 (macOS/MySQL 버전).

```cpp
#include <mysql.h>
#include <string.h>

#ifdef __cplusplus
extern "C" {
#endif

/*
 * 초기화 함수
 */
bool helloworld_init(UDF_INIT *initid, UDF_ARGS *args, char *message)
{
    if (args->arg_count != 0)
    {
        strcpy(message, "helloworld() requires no arguments");
        return 1;
    }

    initid->maybe_null = 0;
    initid->const_item = 1;
    initid->max_length = 29;

    return 0;
}

/*
 * 메인 함수
 */
char *helloworld(UDF_INIT *initid,
                 UDF_ARGS *args,
                 char *result,
                 unsigned long *length,
                 char *is_null,
                 char *error)
{
    const char *msg = "Hello World Mysql Plugin-UDF";
    *length = strlen(msg);
    memcpy(result, msg, *length);
    return result;
}

/*
 * 종료 함수
 */
void helloworld_deinit(UDF_INIT *initid)
{
    /* nothing to free */
}

#ifdef __cplusplus
}
#endif
```

#### `helloworld_init()` 분석

초기화 함수는 **성공 시 0(false), 실패 시 1(true)** 을 반환합니다.

- `args->arg_count != 0`: 이 UDF는 인자를 받지 않습니다. 인자가 있으면 `message`에 오류 문자열을 복사하고 1을 반환합니다.
- `initid->maybe_null = 0`: 이 함수는 절대 NULL을 반환하지 않습니다.
- `initid->const_item = 1`: 입력에 관계없이 항상 같은 값을 반환합니다(상수 함수). MySQL 옵티마이저가 쿼리를 최적화할 수 있습니다.
- `initid->max_length = 29`: 반환 문자열의 최대 길이. `"Hello World Mysql Plugin-UDF"`는 28자이므로 null terminator 포함 29.

#### `helloworld()` 분석

메인 함수는 각 행마다 호출됩니다.

- `result`: MySQL이 미리 할당해준 버퍼입니다. 짧은 문자열은 이 버퍼를 그대로 사용하면 됩니다.
- `*length`: 반환할 문자열의 실제 길이를 설정합니다.
- `*is_null`: 1로 설정하면 NULL을 반환합니다.
- `*error`: 1로 설정하면 오류를 반환합니다.

> **주의**: `result` 버퍼의 기본 크기는 255바이트입니다. 더 긴 문자열이 필요하면 2장에서 배우는 동적 메모리 할당(`initid->ptr`)을 사용해야 합니다.

#### `helloworld_deinit()` 분석

이 예제에서는 `_init()`에서 동적 메모리를 할당하지 않았으므로 해제할 것이 없습니다. 하지만 함수 자체는 반드시 정의해야 합니다. MySQL이 이 함수를 항상 호출하기 때문입니다.
