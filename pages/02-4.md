# 02-4. Linux/MariaDB 버전 비교

Linux MariaDB 버전의 코드는 약간 다릅니다.

```cpp
#include <mysql.h>
#include <string.h>
#include <stdio.h>

extern "C" {
    char* hello_world(UDF_INIT* initid, UDF_ARGS* args, char* result,
                      unsigned long* length, char* is_null, char* error) {
        if (args->arg_count != 1 || args->arg_type[0] != STRING_RESULT) {
            strcpy(result, "Usage: hello_world('string')");
            *length = strlen(result);
            return result;
        }

        const char* input = args->args[0];
        unsigned long input_len = args->lengths[0];

        sprintf(result, "Hello World, %.*s!", (int)input_len, input);
        *length = strlen(result);
        *is_null = 0;
        return result;
    }

    my_bool hello_world_init(UDF_INIT* initid, UDF_ARGS* args, char* message) {
        if (args->arg_count != 1 || args->arg_type[0] != STRING_RESULT) {
            strcpy(message, "hello_world requires exactly one string argument");
            return 1;
        }
        initid->max_length = 255;
        return 0;
    }

    void hello_world_deinit(UDF_INIT* initid) {}
}
```

### macOS/MySQL vs Linux/MariaDB 비교

| 항목 | macOS/MySQL 버전 | Linux/MariaDB 버전 |
|------|----------------|-------------------|
| 반환 타입 (`_init`) | `bool` | `my_bool` |
| 오류 시 메모리 | `initid->ptr` 동적 할당 | `result` 버퍼 직접 사용 |
| 반환 문자열 | `"Welcome {name}, HelloWorld Mysql Plugin"` | `"Hello World, {name}!"` |
| 버퍼 전략 | 동적 할당으로 가변 길이 처리 | `result` 고정 버퍼 사용 |

`my_bool`은 MariaDB에서 정의된 타입으로 MySQL 8.0의 `bool`과 동일하게 동작합니다.

Linux 버전이 `result` 버퍼를 직접 사용하는 것은 간단한 예제이기 때문입니다. 실제 운영 코드에서는 입력 길이에 따라 버퍼 오버플로가 생길 수 있으므로, macOS 버전처럼 동적 할당을 사용하는 것이 더 안전합니다.
