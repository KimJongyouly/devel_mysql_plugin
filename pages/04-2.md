# 04-2. 플러그인 등록 구조

### plugin.h의 플러그인 descriptor

데몬 플러그인은 `mysql/plugin.h`에 정의된 매크로와 구조체를 사용합니다.

MySQL 8.0에서는 `mysql_declare_plugin`, MariaDB에서는 `maria_declare_plugin` 매크로를 사용합니다.

**MariaDB 버전 (Linux):**

```cpp
struct st_mysql_daemon os_logger_info = { MYSQL_DAEMON_INTERFACE_VERSION };

maria_declare_plugin(os_logger)
{
    MYSQL_DAEMON_PLUGIN,    // 플러그인 타입
    &os_logger_info,        // 타입별 인터페이스 구조체
    "os_logger",            // 플러그인 이름 (INSTALL PLUGIN에서 사용)
    "youly",                // 작성자
    "OS Resource Logger",   // 설명
    PLUGIN_LICENSE_GPL,     // 라이선스
    os_logger_init,         // 초기화 함수 (INSTALL PLUGIN 시 호출)
    os_logger_deinit,       // 종료 함수 (UNINSTALL PLUGIN 시 호출)
    0x0100,                 // 버전 (1.0)
    NULL,                   // SHOW STATUS 변수 (없으면 NULL)
    NULL,                   // 시스템 변수 (없으면 NULL)
    "1.0",                  // 버전 문자열 (MariaDB 전용)
    MariaDB_PLUGIN_MATURITY_STABLE  // 안정성 등급 (MariaDB 전용)
}
maria_declare_plugin_end;
```

`MYSQL_DAEMON_INTERFACE_VERSION`은 현재 MySQL/MariaDB API 버전을 나타내는 상수입니다. 빌드 시 사용한 헤더의 버전과 설치된 서버의 버전이 맞지 않으면 `INSTALL PLUGIN` 시 오류가 발생합니다.
