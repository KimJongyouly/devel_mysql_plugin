# 제2장: 인자를 받는 UDF — Hello World with Args

---

## 2.1 UDF 인자 처리 구조

1장에서 만든 `helloworld()`는 인자를 받지 않는 함수였습니다. 실용적인 UDF는 대부분 컬럼 값이나 상수를 인자로 받아 처리합니다. 이 장에서는 문자열 인자를 하나 받아 인사말을 만드는 UDF를 작성합니다.

### UDF_ARGS 구조체

인자 정보는 모두 `UDF_ARGS` 구조체에 담겨 전달됩니다.

```c
typedef struct st_udf_args {
    unsigned int    arg_count;     // 전달된 인자의 개수
    enum Item_result *arg_type;    // 각 인자의 타입 배열
    char           **args;         // 각 인자의 값 포인터 배열
    unsigned long  *lengths;       // 각 인자의 길이 배열
    char           *maybe_null;    // 각 인자가 NULL일 수 있는지
    char          **attributes;    // 인자 이름 (별칭 포함)
    unsigned long  *attribute_lengths; // 인자 이름 길이
    void           *extension;
} UDF_ARGS;
```

`args`는 배열이므로 `args->args[0]`이 첫 번째 인자의 값, `args->args[1]`이 두 번째 인자의 값입니다.

### 인자 타입

`arg_type` 배열의 각 원소는 다음 중 하나입니다.

| 타입 상수 | 설명 | C 타입 |
|----------|------|--------|
| `STRING_RESULT` | 문자열 (VARCHAR, TEXT 등) | `char*` |
| `INT_RESULT` | 정수 (INT, BIGINT 등) | `long long*` |
| `REAL_RESULT` | 실수 (FLOAT, DOUBLE 등) | `double*` |
| `DECIMAL_RESULT` | 고정소수점 (DECIMAL) | `char*` (문자열 표현) |
| `ROW_RESULT` | 다중 컬럼 (특수 용도) | — |

문자열 타입의 경우, `args->args[i]`가 `char*` 포인터이고 `args->lengths[i]`가 실제 바이트 길이입니다. 문자열은 null-terminate되지 않을 수 있으므로 **반드시 `lengths[i]`를 사용**해야 합니다.

```cpp
// 올바른 방법
const char *name = args->args[0];
unsigned long name_len = args->lengths[0];

// 틀린 방법 (null terminate 보장 없음)
const char *name = args->args[0];
size_t len = strlen(name);  // 위험!
```

### NULL 처리

인자가 NULL이면 `args->args[i]`가 `nullptr`입니다.

```cpp
if (args->args[0] == NULL) {
    *is_null = 1;
    return NULL;
}
```

NULL 입력 시 NULL을 반환하는 것이 일반적인 MySQL 함수의 관습입니다. `initid->maybe_null = 1`로 설정해 이 함수가 NULL을 반환할 수 있음을 MySQL에게 알립니다.

## 2.2 동적 메모리 관리

### result 버퍼의 한계

1장에서는 MySQL이 제공하는 `result` 버퍼에 직접 결과를 썼습니다. 이 버퍼의 기본 크기는 **255바이트**입니다. 인자의 길이에 따라 반환 문자열이 달라지는 UDF에서는 255바이트를 초과할 수 있습니다.

예를 들어 이름이 길면:

```
"Welcome 아주아주아주긴이름입니다, HelloWorld Mysql Plugin"
 = 8 + 이름길이 + 25 + 1 바이트
```

이름이 250자라면 총 284바이트가 필요합니다. `result` 버퍼에 쓰면 버퍼 오버플로가 발생합니다.

### initid->ptr 활용

`UDF_INIT`의 `ptr` 필드는 초기화 함수에서 동적으로 할당한 메모리의 포인터를 저장하는 용도입니다.

```
_init()  → malloc(buf_len) → initid->ptr에 저장
메인()   → initid->ptr에 결과 작성 → initid->ptr 반환
_deinit() → free(initid->ptr)
```

이 패턴을 따르면 가변 길이 결과를 안전하게 처리할 수 있습니다.

## 2.3 구현: `helloworld(name)` 함수 (macOS/MySQL)

### 전체 코드

```cpp
#include <mysql.h>
#include <string.h>
#include <stdio.h>
#include <stdlib.h>

#ifdef __cplusplus
extern "C" {
#endif

bool helloworld_init(UDF_INIT *initid, UDF_ARGS *args, char *message)
{
    // 인자가 1개이고 문자열 타입이어야 합니다
    if (args->arg_count != 1 || args->arg_type[0] != STRING_RESULT)
    {
        strcpy(message, "helloworld() requires exactly one string argument (name)");
        return 1;
    }

    initid->maybe_null = 1;   // NULL 반환 가능
    initid->const_item = 0;   // 행마다 다른 결과
    initid->max_length = 300;
    initid->ptr = NULL;       // 아직 할당 안 함

    return 0;
}

char *helloworld(UDF_INIT *initid,
                 UDF_ARGS *args,
                 char *result,
                 unsigned long *length,
                 char *is_null,
                 char *error)
{
    // NULL 입력 처리
    if (args->args[0] == NULL)
    {
        *is_null = 1;
        return NULL;
    }

    const char *name = args->args[0];
    unsigned long name_len = args->lengths[0];

    // 필요한 버퍼 크기 계산
    // "Welcome " (8) + name + ", HelloWorld Mysql Plugin" (25) + null (1)
    unsigned long buf_len = 8 + name_len + 25 + 1;

    // 이전 호출에서 할당된 버퍼가 있으면 해제
    if (initid->ptr)
        free(initid->ptr);

    // 새 버퍼 할당
    initid->ptr = (char *)malloc(buf_len);
    if (!initid->ptr)
    {
        *error = 1;
        return NULL;
    }

    // 결과 문자열 조합
    *length = (unsigned long)snprintf(initid->ptr, buf_len,
                                      "Welcome %.*s, HelloWorld Mysql Plugin",
                                      (int)name_len, name);

    return initid->ptr;
}

void helloworld_deinit(UDF_INIT *initid)
{
    if (initid->ptr)
    {
        free(initid->ptr);
        initid->ptr = NULL;
    }
}

#ifdef __cplusplus
}
#endif
```

### 코드 상세 분석

#### `_init()`: 인자 검증

```cpp
if (args->arg_count != 1 || args->arg_type[0] != STRING_RESULT)
{
    strcpy(message, "helloworld() requires exactly one string argument (name)");
    return 1;
}
```

`_init()`에서 인자를 검증하는 것이 핵심입니다. 인자 검증을 `_init()`에서 하면, 쿼리가 시작될 때 한 번만 검사합니다. 반면 메인 함수에서 하면 수백만 행마다 반복 검사해야 합니다.

`args->arg_type[0]`은 첫 번째 인자의 타입입니다. `STRING_RESULT`를 강제함으로써 정수나 실수가 잘못 전달되는 것을 막습니다.

> **타입 강제 변환**: `args->arg_type[i]`에 원하는 타입을 직접 대입하면 MySQL이 자동으로 변환해줍니다.
> ```cpp
> args->arg_type[0] = STRING_RESULT; // 정수가 오면 문자열로 변환
> ```
> 이 방법을 쓰면 다양한 타입을 받을 수 있습니다.

#### 메인 함수: 버퍼 크기 계산

```cpp
unsigned long buf_len = 8 + name_len + 25 + 1;
```

이 계산에서 정확히 필요한 크기만 할당합니다.

- `8` = `"Welcome "` (공백 포함 8자)
- `name_len` = `args->lengths[0]` (바이트 단위 길이, UTF-8 멀티바이트 가능)
- `25` = `", HelloWorld Mysql Plugin"` (25자)
- `1` = null terminator

#### `snprintf`로 안전한 문자열 조합

```cpp
*length = (unsigned long)snprintf(initid->ptr, buf_len,
                                  "Welcome %.*s, HelloWorld Mysql Plugin",
                                  (int)name_len, name);
```

`%.*s`는 정밀도를 동적으로 지정하는 형식입니다. `*` 자리에 정수(`name_len`)를 전달하면 정확히 그 길이만큼만 출력합니다. 이는 null terminate되지 않은 문자열을 안전하게 처리하는 관용적인 방법입니다.

`snprintf`는 실제로 쓴 바이트 수를 반환합니다. 이를 `*length`에 저장합니다.

#### `_deinit()`: 메모리 해제

```cpp
void helloworld_deinit(UDF_INIT *initid)
{
    if (initid->ptr)
    {
        free(initid->ptr);
        initid->ptr = NULL;
    }
}
```

`_deinit()`은 쿼리가 완전히 끝난 후 딱 한 번 호출됩니다. 이 시점에 `initid->ptr`을 해제합니다.

`NULL` 체크 후 해제하고 다시 `NULL`로 초기화하는 것은 이중 해제(double free) 버그를 방지하는 방어적 코딩입니다.

## 2.4 Linux/MariaDB 버전 비교

Linux MariaDB 버전의 코드는 약간 다릅니다.

```cpp
#include <mysql.h>
#include <string.h>
#include <stdio.h>

extern "C" {
    char* hello_world(UDF_INIT* initid, UDF_ARGS* args, char* result,
                      unsigned long* length, char* is_null, char* error) {
        if (args->arg_count != 1 || args->arg_type[0] != STRING_RESULT) {
            strcpy(result, "Usage: hello_world('string')");
            *length = strlen(result);
            return result;
        }

        const char* input = args->args[0];
        unsigned long input_len = args->lengths[0];

        sprintf(result, "Hello World, %.*s!", (int)input_len, input);
        *length = strlen(result);
        *is_null = 0;
        return result;
    }

    my_bool hello_world_init(UDF_INIT* initid, UDF_ARGS* args, char* message) {
        if (args->arg_count != 1 || args->arg_type[0] != STRING_RESULT) {
            strcpy(message, "hello_world requires exactly one string argument");
            return 1;
        }
        initid->max_length = 255;
        return 0;
    }

    void hello_world_deinit(UDF_INIT* initid) {}
}
```

차이점을 정리하면 다음과 같습니다.

| 항목 | macOS/MySQL 버전 | Linux/MariaDB 버전 |
|------|----------------|-------------------|
| 반환 타입 (`_init`) | `bool` | `my_bool` |
| 오류 시 메모리 | `initid->ptr` 동적 할당 | `result` 버퍼 직접 사용 |
| 반환 문자열 | `"Welcome {name}, HelloWorld Mysql Plugin"` | `"Hello World, {name}!"` |
| 버퍼 전략 | 동적 할당으로 가변 길이 처리 | `result` 고정 버퍼 사용 |

`my_bool`은 MariaDB에서 정의된 타입으로 MySQL 8.0의 `bool`과 동일하게 동작합니다. MySQL 8.0에서는 `bool`을 직접 사용합니다.

Linux 버전이 `result` 버퍼를 직접 사용하는 것은 간단한 예제이기 때문입니다. 실제 운영 코드에서는 입력 길이에 따라 버퍼 오버플로가 생길 수 있으므로, macOS 버전처럼 동적 할당을 사용하는 것이 더 안전합니다.

## 2장 정리

- `UDF_ARGS` 구조체로 인자 개수(`arg_count`), 타입(`arg_type`), 값(`args`), 길이(`lengths`)에 접근한다.
- 인자 검증은 성능을 위해 메인 함수가 아닌 `_init()`에서 수행한다.
- 문자열 인자는 null-terminate가 보장되지 않으므로 `lengths[i]`와 `%.*s`를 사용한다.
- 가변 길이 반환값은 `initid->ptr`에 `malloc()`으로 할당하고, `_deinit()`에서 `free()`한다.
- NULL 입력은 `args->args[i] == NULL`로 확인하고, `*is_null = 1`로 NULL을 반환한다.
