## 1.1 UDF란 무엇인가

### MySQL의 한계와 확장성

MySQL은 `COUNT()`, `SUM()`, `AVG()` 같은 내장 함수를 기본으로 제공합니다. 하지만 업무 현장에서는 이것만으로 해결되지 않는 요구사항이 종종 생깁니다.

예를 들어 이런 상황을 생각해봅시다.

- 사내 암호화 규칙에 따라 특정 컬럼 값을 변환해야 한다
- 외부 API를 호출해 반환값을 SQL 결과로 받고 싶다
- 중앙값, 최빈값 같은 통계 함수가 필요하다
- 머신러닝 모델의 추론 결과를 SQL 쿼리 안에서 쓰고 싶다

이런 경우 애플리케이션 코드에서 행마다 처리하면 수백만 건의 데이터를 네트워크로 전송해야 합니다. 반면 **UDF(User Defined Function)** 를 사용하면 그 처리를 MySQL 서버 안에서 직접 수행할 수 있습니다.

### UDF의 정의

UDF는 C 또는 C++로 작성한 공유 라이브러리(`.so` / `.dylib`)를 MySQL에 등록해, SQL 함수처럼 호출하는 확장 메커니즘입니다.

```sql
SELECT my_function(column) FROM table;
```

이 한 줄이 실제로는 여러분이 C++로 작성한 코드를 MySQL 프로세스 내부에서 직접 실행합니다.

### UDF의 종류

MySQL 플러그인 개발에서 다루는 함수 확장은 크게 세 가지로 나뉩니다.

| 종류 | 설명 | 예 |
|------|------|-----|
| **스칼라 UDF** | 행마다 하나의 값을 반환 | `my_encrypt(col)` |
| **집계 UDF** | 여러 행을 묶어 하나의 값을 반환, `GROUP BY` 지원 | `my_median(col)` |
| **스토리지 엔진** | 테이블 저장·조회 방식 자체를 구현 | `ENGINE=READ_PARQUET` |

이 책에서는 이 세 가지를 단계적으로 다룹니다. 먼저 가장 단순한 스칼라 UDF부터 시작합니다.

### UDF가 적합한 상황

UDF를 고려할 만한 상황은 다음과 같습니다.

- **성능**: 수백만 건 이상의 행을 변환·필터링할 때 애플리케이션보다 DB 내부 처리가 빠르다
- **재사용성**: 여러 애플리케이션·언어에서 동일한 로직을 DB 함수로 공유한다
- **통일성**: 암호화, 마스킹, 해시 등의 로직을 DB에서 한 곳에서 관리한다

반면 UDF가 적합하지 않은 상황도 있습니다. UDF 코드의 버그는 MySQL 서버 프로세스 전체를 크래시시킬 수 있습니다. 따라서 외부 API 호출, 파일 I/O, 복잡한 락 등 위험한 작업은 UDF보다 저장 프로시저나 애플리케이션 계층에서 처리하는 것이 안전합니다.

### 헤더 파일 확인

`mysql.h` 를 열어보면 UDF 개발에 필요한 구조체들이 정의되어 있습니다.

```c
// mysql.h 내 UDF 관련 핵심 타입
typedef struct st_udf_init {
    my_bool   maybe_null;    // NULL 반환 가능 여부
    unsigned int  decimals;  // 소수점 자릿수 (REAL 반환시)
    unsigned long max_length; // 최대 반환 길이
    char     *ptr;           // 초기화 함수에서 할당한 메모리 포인터
    my_bool   const_item;    // 입력이 상수면 1
    void     *extension;
} UDF_INIT;

typedef struct st_udf_args {
    unsigned int  arg_count;    // 인자 개수
    enum Item_result *arg_type; // 각 인자의 타입
    char        **args;         // 각 인자의 값
    unsigned long *lengths;     // 각 인자의 길이
    char        *maybe_null;    // 각 인자의 NULL 가능 여부
    // ...
} UDF_ARGS;
```

이 두 구조체는 앞으로 모든 UDF 코드에서 반복해서 만나게 됩니다.

## 1.3 첫 번째 UDF 작성

### UDF의 세 가지 함수

MySQL 스칼라 UDF는 세 개의 C 함수로 구성됩니다. 함수 이름은 **UDF 이름을 접두어**로 합니다.

| 함수 | 시그니처 | 호출 시점 |
|------|---------|---------|
| `{name}_init()` | `bool f(UDF_INIT*, UDF_ARGS*, char*)` | 쿼리 실행 시작 시 **1회** |
| `{name}()` | `char* f(UDF_INIT*, UDF_ARGS*, char*, ulong*, char*, char*)` | **행마다** |
| `{name}_deinit()` | `void f(UDF_INIT*)` | 쿼리 완료 후 **1회** |

MySQL이 UDF를 호출할 때 이 순서는 항상 고정되어 있습니다.

```
쿼리 시작 → helloworld_init() 1회
              ↓
행 1     → helloworld()
행 2     → helloworld()
행 3     → helloworld()
              ↓
쿼리 종료 → helloworld_deinit() 1회
```

### extern "C" 선언

C++로 UDF를 작성할 때는 `extern "C"` 블록이 필수입니다. C++는 함수 오버로딩을 지원하기 위해 함수 이름을 내부적으로 변환(name mangling)합니다. 하지만 MySQL은 `.so` 파일에서 C 이름 규칙으로 함수를 찾습니다. `extern "C"` 없이 컴파일하면 MySQL이 함수를 찾지 못해 등록이 실패합니다.

```cpp
#ifdef __cplusplus
extern "C" {
#endif

// UDF 함수들은 여기에

#ifdef __cplusplus
}
#endif
```

`#ifdef __cplusplus` 조건부 컴파일을 사용하면 C와 C++ 양쪽에서 같은 소스를 컴파일할 수 있습니다.

### 전체 코드 설명

아래는 `helloworld` UDF의 전체 소스코드입니다 (macOS/MySQL 버전).

```cpp
#include <mysql.h>
#include <string.h>

#ifdef __cplusplus
extern "C" {
#endif

/*
 * 초기화 함수
 */
bool helloworld_init(UDF_INIT *initid, UDF_ARGS *args, char *message)
{
    if (args->arg_count != 0)
    {
        strcpy(message, "helloworld() requires no arguments");
        return 1;
    }

    initid->maybe_null = 0;
    initid->const_item = 1;
    initid->max_length = 29;

    return 0;
}

/*
 * 메인 함수
 */
char *helloworld(UDF_INIT *initid,
                 UDF_ARGS *args,
                 char *result,
                 unsigned long *length,
                 char *is_null,
                 char *error)
{
    const char *msg = "Hello World Mysql Plugin-UDF";
    *length = strlen(msg);
    memcpy(result, msg, *length);
    return result;
}

/*
 * 종료 함수
 */
void helloworld_deinit(UDF_INIT *initid)
{
    /* nothing to free */
}

#ifdef __cplusplus
}
#endif
```

#### `helloworld_init()` 분석

```cpp
bool helloworld_init(UDF_INIT *initid, UDF_ARGS *args, char *message)
```

초기화 함수는 **성공 시 0(false), 실패 시 1(true)** 을 반환합니다.

- `args->arg_count != 0`: 이 UDF는 인자를 받지 않습니다. 인자가 있으면 `message`에 오류 문자열을 복사하고 1을 반환합니다. MySQL은 이 메시지를 오류로 표시합니다.
- `initid->maybe_null = 0`: 이 함수는 절대 NULL을 반환하지 않습니다.
- `initid->const_item = 1`: 입력에 관계없이 항상 같은 값을 반환합니다(상수 함수). MySQL 옵티마이저가 이를 알면 쿼리를 최적화할 수 있습니다.
- `initid->max_length = 29`: 반환 문자열의 최대 길이. `"Hello World Mysql Plugin-UDF"`는 28자이므로 null terminator 포함 29.

#### `helloworld()` 분석

```cpp
char *helloworld(UDF_INIT *initid,
                 UDF_ARGS *args,
                 char *result,
                 unsigned long *length,
                 char *is_null,
                 char *error)
```

메인 함수는 각 행마다 호출됩니다. 반환 타입이 `char *`인 것은 문자열을 반환하기 때문입니다.

- `result`: MySQL이 미리 할당해준 버퍼입니다. 짧은 문자열은 이 버퍼를 그대로 사용하면 됩니다.
- `*length`: 반환할 문자열의 실제 길이를 설정합니다.
- `*is_null`: 1로 설정하면 NULL을 반환합니다.
- `*error`: 1로 설정하면 오류를 반환합니다.

`memcpy(result, msg, *length)` 로 MySQL의 버퍼에 문자열을 복사하고 `result`를 반환합니다.

> **주의**: `result` 버퍼의 기본 크기는 255바이트입니다. 더 긴 문자열이 필요하면 2장에서 배우는 동적 메모리 할당(`initid->ptr`)을 사용해야 합니다.

#### `helloworld_deinit()` 분석

```cpp
void helloworld_deinit(UDF_INIT *initid)
{
    /* nothing to free */
}
```

이 예제에서는 `_init()`에서 동적 메모리를 할당하지 않았으므로 해제할 것이 없습니다. 하지만 함수 자체는 반드시 정의해야 합니다. MySQL이 이 함수를 항상 호출하기 때문입니다.

## 1.4 빌드와 설치

### Makefile 구조

```makefile
CXX      = g++
TARGET   = helloworld_udf.so
SRC      = helloworld_udf.cpp

MYSQL_CONFIG  = /usr/local/mysql/bin/mysql_config
MYSQL_CFLAGS := $(shell $(MYSQL_CONFIG) --cflags)
MYSQL_PLUGINDIR := $(shell $(MYSQL_CONFIG) --plugindir)

CXXFLAGS = $(MYSQL_CFLAGS) -fPIC -O2 -Wall

UNAME := $(shell uname)
ifeq ($(UNAME), Darwin)
    SHARED_FLAG = -dynamiclib
else
    SHARED_FLAG = -shared
endif

all: $(TARGET)

$(TARGET): $(SRC)
	$(CXX) $(SHARED_FLAG) $(CXXFLAGS) -o $@ $<

install: $(TARGET)
	cp $(TARGET) $(MYSQL_PLUGINDIR)/
```

핵심 컴파일 플래그를 이해해 봅시다.

- **`-fPIC`** (Position Independent Code): 공유 라이브러리는 메모리의 어느 주소에 로드될지 모릅니다. `-fPIC`는 절대 주소 대신 상대 주소를 사용하는 코드를 생성합니다. 공유 라이브러리에는 **반드시** 필요합니다.
- **`-dynamiclib` (macOS) / `-shared` (Linux)**: 실행 파일이 아닌 동적 라이브러리(`.so` / `.dylib`)를 생성합니다.
- **`$(MYSQL_CONFIG) --cflags`**: `mysql_config`가 MySQL 헤더 경로(`-I/path/to/include`)를 자동으로 제공합니다. MySQL 버전마다 경로가 다를 수 있어, 하드코딩 대신 이 방법을 사용합니다.

### 빌드 실행

```bash
cd 01_HelloWorld/Mac   # 또는 01_HelloWorld/Linux
make
```

성공하면 `helloworld_udf.so`(macOS: `.dylib`) 파일이 생성됩니다.

### 플러그인 디렉터리에 설치

```bash
sudo make install
```

또는 수동으로:

```bash
# MySQL 플러그인 디렉터리 확인
mysql_config --plugindir
# 예: /usr/local/mysql/lib/plugin

cp helloworld_udf.so /usr/local/mysql/lib/plugin/
```

Linux MariaDB에서는:

```bash
sudo cp udf_hello_world.so /home/mariadb/mariadb_11.4.3/lib/plugin/
sudo chown mariadb:mariadb /home/mariadb/mariadb_11.4.3/lib/plugin/udf_hello_world.so
sudo chmod 755 /home/mariadb/mariadb_11.4.3/lib/plugin/udf_hello_world.so
```

파일을 복사하는 것만으로는 MySQL에 등록되지 않습니다. 다음 단계에서 SQL 명령으로 등록해야 합니다.

### MySQL에 함수 등록

MySQL 클라이언트에 접속 후:

```sql
CREATE FUNCTION helloworld RETURNS STRING SONAME 'helloworld_udf.so';
```

- `RETURNS STRING`: 반환 타입. `STRING`, `INTEGER`, `REAL`, `DECIMAL` 중 선택합니다.
- `SONAME 'helloworld_udf.so'`: 플러그인 디렉터리에 있는 파일 이름입니다.

MySQL은 이 명령을 받으면 플러그인 디렉터리에서 해당 파일을 열고, `helloworld_init`, `helloworld`, `helloworld_deinit` 세 함수가 존재하는지 확인합니다. 모두 있으면 등록에 성공합니다.

등록 정보는 `mysql.func` 테이블에 저장되어 MySQL 재시작 후에도 유지됩니다.

```sql
SELECT * FROM mysql.func WHERE name = 'helloworld';
```

## 1.5 동작 확인

### 기본 호출

```sql
SELECT helloworld();
-- 결과:
-- +------------------------------+
-- | helloworld()                 |
-- +------------------------------+
-- | Hello World Mysql Plugin-UDF |
-- +------------------------------+
```

### 테이블과 함께 사용

이 UDF는 행마다 같은 값을 반환하므로, 테이블과 함께 사용하면 모든 행에 동일한 값이 나옵니다.

```sql
CREATE TABLE t (id INT);
INSERT INTO t VALUES (1), (2), (3);

SELECT id, helloworld() FROM t;
-- +----+------------------------------+
-- | id | helloworld()                 |
-- +----+------------------------------+
-- |  1 | Hello World Mysql Plugin-UDF |
-- |  2 | Hello World Mysql Plugin-UDF |
-- |  3 | Hello World Mysql Plugin-UDF |
-- +----+------------------------------+
```

MySQL은 `initid->const_item = 1` 설정을 알고 있기 때문에, 실제로는 `helloworld()` 를 한 번만 호출하고 결과를 재사용하는 최적화를 수행할 수 있습니다.

### 잘못된 호출 확인

인자를 전달하면 `_init()` 에서 설정한 오류 메시지가 반환됩니다.

```sql
SELECT helloworld('test');
-- ERROR 1123 (HY000): Can't initialize function 'helloworld';
-- helloworld() requires no arguments
```

### 함수 제거

```sql
DROP FUNCTION IF EXISTS helloworld;
```

### 트러블슈팅

**`mysql.h: No such file or directory`**

```
helloworld_udf.cpp:1:10: fatal error: mysql.h: No such file or directory
```

MySQL 헤더가 없거나 `mysql_config` 경로가 올바르지 않습니다.

```bash
# mysql_config 위치 확인
which mysql_config
/usr/local/mysql/bin/mysql_config   # macOS 기본

# Makefile의 MYSQL_CONFIG 경로를 수정합니다
```

**`Can't open shared library 'helloworld_udf.so'`**

플러그인 파일이 올바른 디렉터리에 없거나 권한이 맞지 않습니다.

```sql
-- 플러그인 디렉터리 확인
SHOW VARIABLES LIKE 'plugin_dir';
```

해당 경로에 `.so` 파일이 있는지 확인하고, 읽기 권한이 있는지 점검합니다.

**`function 'helloworld' already exists`**

이미 등록된 함수를 다시 등록하려 했습니다.

```sql
DROP FUNCTION IF EXISTS helloworld;
-- 재등록
CREATE FUNCTION helloworld RETURNS STRING SONAME 'helloworld_udf.so';
```

플러그인 파일을 교체한 경우에도 반드시 `DROP FUNCTION` 후 재등록이 필요합니다. MySQL은 한 번 로드한 라이브러리를 캐시하기 때문에, 파일만 교체해도 새 코드가 즉시 반영되지 않습니다.

## 1장 정리

이 장에서 배운 내용을 정리합니다.

- UDF는 C/C++로 작성한 공유 라이브러리를 MySQL에 등록해 SQL 함수로 사용하는 확장 메커니즘이다.
- 스칼라 UDF는 `_init()`, 메인 함수, `_deinit()` 세 가지 C 함수로 구성된다.
- C++로 작성할 때는 `extern "C"` 선언이 필수다.
- `-fPIC` 플래그와 `-shared`(`-dynamiclib`) 플래그로 공유 라이브러리를 빌드한다.
- `CREATE FUNCTION ... SONAME '...'` 으로 등록하고, `DROP FUNCTION` 으로 제거한다.

다음 장에서는 인자를 받는 UDF를 작성하며 `UDF_ARGS` 구조체와 동적 메모리 관리를 배웁니다.
